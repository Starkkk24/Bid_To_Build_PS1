# Bug Report (Level 1) - Symptom Analysis

## 1. Confusing Status Updates
* **Symptom:** Cards briefly show as "Printed" immediately after submission, but later switch back to "Processing". This makes it hard to know when a card is actually done.

## 2. Spam Submissions
* **Symptom:** Users can rapidly click the "Generate ID Card" button multiple times, resulting in multiple identical entries appearing in the print queue.

## 3. Missing Images
* **Symptom:** Frequently, newly created ID cards appear in the list, but their profile pictures are broken links or fail to load.

## 4. Stuck Pending Cards
* **Symptom:** Occasionally, cards get permanently stuck in the "Pending" state and never transition to "Processing" or "Printed", despite the UI claiming the upload was successful.

## 5. Misleading Success Messages
* **Symptom:** The system sometimes says "ID Card processed successfully," but the card is never actually processed or printed correctly.

## 6. Unpredictable Speed
* **Symptom:** Sometimes submitting an ID card takes less than a second, while other times it hangs for several seconds before finishing.

## 7. List Doesn't Auto-Update
* **Symptom:** After submitting a new ID card, it doesn't appear in the right-side list until the user manually clicks the refresh button.

## 8. Swapped Text Fields
* **Symptom:** The name and department fields are visually swapped in the status list compared to what was entered in the form.
